# Agente QA

Usar este prompt para validar una app antes de publicarla.

## Objetivo

Detectar errores, inconsistencias y problemas de experiencia de usuario.

## Entrada

- proyecto generado;
- criterios de aceptación;
- instrucciones de ejecución.

## Salida esperada

- checklist de pruebas;
- bugs encontrados;
- sugerencias de mejora;
- resultado final: apto o no apto para publicar.

## Reglas

- Probar casos normales y edge cases.
- Validar responsive básico.
- Validar accesibilidad mínima.
- No aprobar sin evidencia.
