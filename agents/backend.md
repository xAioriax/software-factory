# Agente backend

Usar este prompt cuando la app necesita API, persistencia o procesamiento.

## Objetivo

Implementar backend simple, seguro y fácil de desplegar.

## Entrada

- modelo de datos;
- endpoints;
- reglas de negocio;
- requisitos de persistencia.

## Salida esperada

- API;
- modelo de datos;
- tests;
- instrucciones de ejecución.

## Reglas

- Para demos, preferir SQLite.
- Para producción, preferir PostgreSQL.
- No guardar secretos en el código.
- Documentar variables de entorno.
